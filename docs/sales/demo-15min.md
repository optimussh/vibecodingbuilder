## 스크립트 (말할 말 + 클릭)

### 0:00 — 훅 (1분)

> “요즘 팀원들이 Cursor나 개인 API 키로 바이브코딩을 합니다.  
> 속도는 좋은데, **키가 흩어지고, 누가 어떤 코드를 바꿨는지 안 남고,  
> 인턴 PC에서 사내 레포가 그대로 노출**됩니다.  
> CodeHarbor는 그 속도를 **사내 정책 안으로** 넣는 플랫폼입니다.”

화면에 포털만 띄운 상태.

---

### 1:00 — 포지션 (2분)

포털 카드 가리키며:

> “우리는 IDE를 하나 더 만드는 게 아니라 **게이트웨이**입니다.  
> 로그인 · 워크스페이스 격리 · 모델 키 BFF · 감사 · 쿼터.  
> 에이전트 엔진은 OpenCode, UI는 OpenChamber를 붙입니다.  
> 엔진은 교체 가능하고, **통제 평면이 제품**입니다.”

한 줄 카테고리:

> **Governed Vibe Coding Platform**

---

### 3:00 — 격리 라이브 (3분)  ★ 필수

1. **브라우저 A:** `user1` 로그인 → 포털  
2. **Chamber 자동 오픈** 또는 채팅 UI → user1 워크스페이스 경로 보여주기  
3. 파일 트리/워크스페이스에 `user1-only` 티어 파일이 보이면 강조  
4. **브라우저 B (시크릿):** `user2` 로그인  
5. user2에서 user1 경로·세션 접근 시도 불가 / 다른 워크스페이스만 표시  

말할 말:

> “같은 서버, 같은 OpenCode여도 **directory + session ownership** 으로 갈라집니다.  
> 이게 개인 SaaS AI와 다른 첫 번째 지점입니다.”

실패 시 백업: `/api` isolation 테스트 결과 슬라이드 1장.

---

### 6:00 — 바이브 루프 (4분)

**user1** 으로:

1. 채팅 또는 Chamber에서 세션 열기  
2. 프롬프트 예:

   > “워크스페이스에 `hello-harbor.md` 파일을 만들고, CodeHarbor 한 줄 소개를 한글로 적어줘.”

3. 파일 생성/디프 확인 (Diff 버튼 또는 파일 트리)  
4. (시간 되면) Git status 패널 또는 Preview 버튼 언급  

말할 말:

> “개발자 경험은 익숙한 에이전트 루프입니다.  
> 차이는 **이 모든 호출이 게이트웨이를 통한다**는 점입니다.”

키 없을 때:

> “지금 데모 환경에 키가 없으면 생성 스텝은 스킵하고,  
> 구조상 키는 `.env`/관리자 vault에만 있고 브라우저 네트워크 탭에 안 찍힙니다.”  
> (가능하면 DevTools Network에서 API 응답에 key 없음 보여주기)

---

### 10:00 — 거버넌스 (2.5분)

클릭 순서:

1. **쿼터** — UI 또는 `GET /api/quota` (admin/user 메시지 한도)  
2. **감사** — 파일 `data/audit` 또는 향후 Admin 화면 목업; `workspace.bind`, `message.send` 이벤트 언급  
3. **스택** — `/api/stack` 포트 맵 (운영 가시성)  
4. **BFF** — “Gemini 키는 서버만; `/opencode` 는 로그인 필수”

말할 말:

> “보안 팀 질문 세 가지에 바로 답합니다.  
> 1) 키 어디 있나 → 서버 vault  
> 2) 누가 뭘 했나 → audit  
> 3) 남용 어떻게 막나 → quota + 격리”
